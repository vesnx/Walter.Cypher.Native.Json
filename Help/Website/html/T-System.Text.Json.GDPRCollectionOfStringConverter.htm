GDPRCollectionOfStringConverter Class

Provides a JSON converter for a collection of strings, ensuring GDPR-compliant encryption and decryption of data with a rotating IV for enhanced security. This converter encrypts each string in the collection during serialization and decrypts them during deserialization, making it suitable for handling sensitive information that requires confidentiality.

Remarks

The GDPRCollectionOfStringConverter secures collections of string data by encrypting the values when serialized to JSON and decrypting them back into their original form upon deserialization. The converter uses a specified password along with a rotating initialization vector (IV), which is stored alongside the cypher bytes. This approach ensures that the same plaintext results in a different encrypted output each time, significantly enhancing security.

Rotating IVs make the encryption scheme more robust by preventing certain types of cryptographic attacks and making the encrypted data less predictable. The IV is automatically managed and included in the encrypted data, ensuring seamless encryption and decryption processes without additional setup.

Usage of this converter is especially recommended for applications that process or store personally identifiable information (PII), financial data, or any other types of sensitive information that fall under GDPR or similar privacy regulations.

Examples

This example demonstrates how to use the GDPRCollectionOfStringConverter on a property for JSON serialization.
C#
public class SensitiveData
{
    [JsonConverter(typeof(GDPRCollectionOfStringConverter))]
    public List<string> ConfidentialInformation { get; set; }
}

var sensitiveData = new SensitiveData
{
    ConfidentialInformation = new List<string> { "SensitiveValue1", "SensitiveValue2" }
};

var options = new JsonSerializerOptions
{
    Converters = { new GDPRCollectionOfStringConverter() }
};
var json = JsonSerializer.Serialize(sensitiveData, options);
// Each 'ConfidentialInformation' value is encrypted with a unique IV, resulting in different cypher text for the same plaintext.
